# ARIA — AI Marketing Operating System

ARIA is an AI-native marketing operating system for D2C brands. It helps a marketer decide **which business goal to pursue**, **which shoppers matter**, **what to say**, **which channel to use**, and then tracks what happened after the campaign was sent.

Built for the **Xeno Engineering Internship Assignment 2026**.

## Product Point Of View

The assignment asks for a shopper-engagement CRM, not a sales pipeline CRM. ARIA is built around a fictional D2C fashion brand, **Lumora**, and focuses on revenue moments:

- lapsed shoppers who need a win-back offer
- at-risk shoppers who should be retained before they churn
- VIP shoppers who deserve early access
- new shoppers who need a second-purchase nudge

Instead of starting with a campaign form, ARIA starts with a business goal:

```txt
Increase revenue by 15% without over-discounting
```

ARIA analyzes the customer base, scores revenue risk and opportunity, recommends actions, predicts impact, explains tradeoffs, and waits for human approval before execution.

## Assignment Alignment

| Requirement | How ARIA handles it |
| --- | --- |
| Ingest customers and orders | Seed data plus CSV import for customers and optional order rows |
| Segment shoppers | JSON filter rules generated by AI and translated into Prisma queries |
| Personalized communication | `{{name}}` personalization before dispatch |
| Separate channel service | Express service simulates Email, SMS, WhatsApp, and RCS |
| Async callbacks | Channel service calls CRM `/api/receipt` with lifecycle events |
| Performance insights | Dashboard, campaign analytics, funnel views, and ARIA readouts |
| AI-native UX | Mission Control, goal planning, proactive insights, AI reasoning, predictions, and human approval |

## Key Features

### ARIA Mission Control

- Business Health Score
- Revenue At Risk
- Revenue Opportunity
- Predicted Monthly Revenue
- AI-generated executive summary
- Recommended actions with confidence, reasoning, and tradeoffs
- Goal-driven strategy builder

### Growth Agent

- Natural language to campaign in one flow
- Gemini-powered generation with deterministic fallback responses
- Produces segment rules, message copy, channel recommendation, and reasoning
- Can launch from dashboard insights or existing segments with context prefilled

### Proactive Operating Layer

- Time-aware greeting
- ARIA Insights for lapsing, at-risk, VIP, and first-campaign opportunities
- Recent campaigns and delivery funnel
- CSV import modal for data ingestion
- Pre-launch prediction engine for reach, opens, clicks, conversions, revenue, and ROI

### Customer Intelligence

- Shopper lifecycle states: Active, New, VIP, At Risk, Lapsed
- Search and lifecycle filters
- Next-best-action hints for each shopper
- 200 seeded customers with realistic purchase history
- Customer 360 pages with churn risk, preferred channel, predicted next purchase, AI summary, and engagement memory

### Segments

- AI-generated and manual audience groups
- Live audience counts
- Sample matched shoppers
- One-click campaign creation from any segment

### Campaign Analytics

- Sent, delivered, opened, clicked, converted, failed, queued
- Live auto-refresh
- Per-campaign funnel
- ARIA readout with bottleneck and next best action
- Channel orchestration event stream from simulated provider callbacks

### Channel Service

- Separate Express service
- Simulates provider behavior and channel-specific probabilities
- Sends async callbacks to the CRM receipt API
- Retries failed callbacks once

## Running Locally

```bash
git clone https://github.com/oooo7/aria-crm.git
cd aria-crm

npm install
cp .env.example .env
```

Update `.env`:

```bash
DATABASE_URL="postgresql://USER:PASSWORD@HOST:PORT/DATABASE"
GEMINI_API_KEY="your_google_gemini_api_key"
CHANNEL_SERVICE_URL="http://localhost:3001"
```

Run database setup:

```bash
npx prisma migrate dev
npx prisma db seed
```

Start the CRM:

```bash
npm run dev
```

Start the channel service in a second terminal:

```bash
cd channel-service
npm install
npm run dev
```

The CRM runs on `http://localhost:3000` and the channel service runs on `http://localhost:3001`.

## System Design Tradeoffs

### What I optimized for

- A complete end-to-end shopper engagement loop
- Clear AI-native product experience
- Simple code that can be discussed in an interview
- Realistic async delivery lifecycle without real messaging providers

### What I intentionally kept simple

- Campaign dispatch is synchronous over the selected audience for assignment scope
- Channel callbacks retry once instead of using a durable queue
- Segment rules support the fields needed for the demo rather than a full rule-builder DSL
- Conversion is simulated instead of tied to a real order attribution pipeline

### What I would improve at scale

- BullMQ/Redis queue for campaign dispatch
- Idempotency keys for receipt callbacks
- Dead-letter queue for failed channel callbacks
- Batch sends and rate limits per channel
- Materialized analytics tables for large campaign volumes
- A richer segment query builder with nested AND/OR conditions
- Multi-tenant brand/workspace model

## Walkthrough Demo Flow

1. Open Mission Control and explain Business Health Score, revenue at risk, and AI recommendations.
2. Enter a goal like "Increase revenue by 15%" and show ARIA building a strategy before asking for approval.
3. Open Growth Agent from a recommendation and show audience, channel, copy, reasoning, and predicted impact.
4. Launch the campaign.
5. Open Campaign Analytics and show live delivery/open/click/conversion funnel plus provider event stream.
6. Open Segments to show audience counts and sample matched shoppers.
7. Open Customers and a Customer 360 profile to show lifecycle, churn risk, preferred channel, predicted next purchase, and recommended action.
8. Briefly explain the separate channel service and `/api/receipt` callback loop.
